Discrepancy between local evaluation score and Kaggle Leaderboard

Hi everyone,

How exactly does NeuroGolf calculate the final cost score? Also, does the evaluation pipeline check the correctness of the ONNX tasks using only the train and test splits, or does it include arc-gen and hidden data?

I am consistently noticing a score gap between my local evaluation machine and my Kaggle submissions. For instance, in one run, my local script scores 6431.63, but drops to 6413.53 upon submission. In another case, a local score of 6413.54 actually climbs to 6425.60 on the leaderboard.

Has anyone else encountered this scenario, and what causes these variations?

It's train, test, arg-gen, plus a small hidden test set. There are two main ways this arises.

There is an environment tooling bug around Conv that has been discussed and the fix is known. This can cause score instability. See this thread https://www.kaggle.com/competitions/neurogolf-2026/discussion/699840
Hidden test set failure resulting in a task scoring 0 through overfitting. You can find the bad tasks through bisecting you submission until you find the culprit(s). Then going forward make sure you submit newly solved tasks/batches so you identify them early.
