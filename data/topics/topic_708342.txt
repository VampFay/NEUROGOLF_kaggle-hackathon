Questions About the ONNX Evaluation Environment Versions and Supported Operators

@mmoffitt Dear organizers,

I have recently encountered the error Error processing onnx networks for tasks multiple times when submitting ONNX models. At the moment, the error message is quite general, and without more detailed logs or information about the evaluation environment and compatibility constraints, it is difficult to debug the issue locally.

In addition, since the evaluation system has gone through several updates during the competition, I am not sure whether the environment information mentioned in earlier posts is still up to date. Therefore, I would like to confirm the current evaluation environment details so that participants can generate and validate more compatible ONNX models locally.

I would like to ask the following questions:

What version of onnx is currently used in the evaluation environment? What version of onnxruntime is currently used in the evaluation environment? What is the highest supported ONNX IR version? What is the highest supported opset version? Is there any additional operator whitelist or list of disabled operators?

If possible, I would also like to ask whether more specific error information from the processing stage could be provided without revealing evaluation details, such as which model, node, operator, shape, or dtype check caused the failure. This would make it much easier for us to reproduce and fix such issues locally.

Thank you!
